    Manejo_Errores (project v0.1.0)

    EJEMPLOS

    Ejemplo 1: Manejo de errores con try..rescue

    iex(1)> try do
...(1)>   #raise ArgumentError, message: "ERROR EN LA INICIALIZACION."
...(1)>   raise "ESTE ES EL ERROR DE RUNTIME"
...(1)> rescue
...(1)>   error1 in ArgumentError -> IO.puts("Ocurrio un error: "<> error1.message)
...(1)>   error2 in RuntimeError -> IO.puts("Error de Runtime: "<> error2.message)
...(1)> end
Error de Runtime: ESTE ES EL ERROR DE RUNTIME

    Ejemplo 2: Manejo de errores con After

    iex(2)> try do
...(2)>   raise ArgumentError, message: "ERROR EN LA INICIALIZACION."
...(2)> rescue
...(2)>   error1 in ArgumentError -> IO.puts("Ocurrio un error: "<> error1.message)
...(2)> after
...(2)>   IO.puts " - Sentencia que se ejecuta despues del manejo del error - "
...(2)> end
Ocurrio un error: ERROR EN LA INICIALIZACION.
- Sentencia que se ejecuta despues del manejo del error -

    Ejemplo 3: Creacion de nuestro propio error

    iex(3)> defmodule MiError do
...(3)>   defexception message: "Este es el mensade de mi error"
...(3)> end
...(3)> try do
...(3)>   raise MiError
...(3)> rescue
...(3)>   error in MiError -> IO.inspect error
...(3)> end
%MiError{message: "Este es el mensade de mi error"}

    Ejemplo 4: Manejo de errores con Try..Catch

    iex(4)> try do
...(4)>   for x <- [1,3,4,5,99,6,7,8] do
...(4)>     if (x == 99), do: throw(x)
...(4)>   end
...(4)> catch
...(4)>   x -> IO.puts "Capturado el elemento: #{x}"
...(4)> end
Capturado el elemento: 99

    Ejemplo 5: Utilizacion de Exit

    iex(5)> try do
...(5)>   exit "mensaje de error de la salida"
...(5)> catch
...(5)>   :exit, _ -> IO.puts "ERROR DE LA SALIDA"
...(5)> end
ERROR DE LA SALIDA
